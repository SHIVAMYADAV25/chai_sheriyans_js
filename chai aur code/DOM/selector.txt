üß© 1. getElementById()

üëâ Used to select a single element by its unique id.

<p id="msg">Hello Shivam!</p>
<script>
  const para = document.getElementById('msg');
  console.log(para); // <p id="msg">Hello Shivam!</p>
</script>

üè∑Ô∏è 2. .id and .className

Used to get or set the id or class of an element.

para.id;          // "msg"
para.className;   // ""  (no class yet)

para.className = "highlight"; // set class


Result in DOM:

<p id="msg" class="highlight">Hello Shivam!</p>

üéØ 3. getAttribute() and setAttribute()

Used to access or modify any attribute of an element.

<button id="btn" data-info="clickable">Press</button>
<script>
  const btn = document.getElementById('btn');

  console.log(btn.getAttribute('data-info')); // "clickable"

  btn.setAttribute('title', 'Click Me'); // add new attribute
</script>


Result:

<button id="btn" data-info="clickable" title="Click Me">Press</button>

üé® 4. .style

Used to apply inline CSS styles directly via JS.

btn.style.color = "red";
btn.style.backgroundColor = "yellow";


Result:

<button style="color:red; background-color:yellow;">Press</button>


‚ö†Ô∏è Note: .style modifies inline styles only, not external CSS classes.

üß† 5. .textContent

Gets or sets the raw text inside an element ‚Äî even hidden ones (e.g., display: none).

<div id="textDiv">
  Hello <span style="display:none;">World</span>
</div>

<script>
  const d = document.getElementById('textDiv');
  console.log(d.textContent); // "Hello World"
</script>


‚úîÔ∏è Includes hidden text
‚ùå No formatting (HTML tags ignored)

üßæ 6. .innerText

Gets visible text only (respects CSS visibility and layout).

console.log(d.innerText); // "Hello"


‚úîÔ∏è Excludes hidden text (display:none)
‚úîÔ∏è Includes line breaks visible on page
‚ùå Slower (forces layout reflow)

üí° Difference between innerText vs textContent
Feature	innerText	textContent
Includes hidden text	‚ùå	‚úÖ
Preserves formatting (line breaks)	‚úÖ	‚ùå
Reads visible text only	‚úÖ	‚ùå
Performance	Slower	Faster
Use case	For user-visible text	For raw text extraction
üß± 7. .innerHTML

Gets or sets the HTML content inside an element (parsed as HTML).

<div id="box"><p>Hello</p></div>
<script>
  const box = document.getElementById('box');
  console.log(box.innerHTML); // "<p>Hello</p>"

  box.innerHTML = "<b>Hi Shivam!</b>"; // replace content
</script>


Result:

<div id="box"><b>Hi Shivam!</b></div>


‚ö†Ô∏è Note: Replaces all inside content (not safe if inserting user input ‚Üí XSS risk).

üß© 8. .outerHTML

Includes the element itself + its inner HTML.

console.log(box.outerHTML);
// "<div id='box'><b>Hi Shivam!</b></div>"


If you reassign it, the element itself gets replaced:

box.outerHTML = "<section>New</section>";

üß≠ 9. getElementsByClassName()

Selects all elements with a given class ‚Üí returns HTMLCollection.

<p class="fruit">Apple</p>
<p class="fruit">Mango</p>

<script>
  const fruits = document.getElementsByClassName('fruit');
  console.log(fruits); // HTMLCollection(2)
  console.log(fruits[0].textContent); // Apple
</script>


‚ö†Ô∏è Live collection ‚Äî updates automatically if DOM changes.

üîç 10. querySelector() and querySelectorAll()

Select elements using CSS selectors ‚Äî the most flexible method.

<div class="item">One</div>
<div class="item">Two</div>

<script>
  const firstItem = document.querySelector('.item');
  console.log(firstItem.textContent); // "One"

  const allItems = document.querySelectorAll('.item');
  console.log(allItems); // NodeList(2)
</script>


‚úÖ querySelector() ‚Üí returns first match
‚úÖ querySelectorAll() ‚Üí returns NodeList (can use forEach())

üßÆ 11. NodeList vs HTMLCollection
Feature	NodeList	HTMLCollection
Returned by	querySelectorAll()	getElementsByClassName(), getElementsByTagName()
Type	Static (not live)	Live (auto-updates)
forEach() supported	‚úÖ	‚ùå (in some browsers)
Includes text nodes	Sometimes	Only element nodes
Conversion to array	Array.from(list)	Array.from(collection)
